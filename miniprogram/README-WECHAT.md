# 店评小帮 · 微信小程序（原生）操作与上架指南

本目录为**微信原生小程序**工程，与仓库根目录 Next.js 网页共用同一套后端接口（`/api/generate-replies`、`/api/extract-review-from-image`）。小程序端**无登录、无用户信息采集**，收藏数据仅存本机。

---

## 一、目录结构

```
miniprogram/
├── app.js                 # 全局入口，配置 apiBase
├── app.json / app.wxss
├── project.config.json    # 开发者工具项目配置（需改 appid）
├── sitemap.json
├── utils/
│   ├── api.js             # wx.request 封装
│   ├── constants.js       # 品类/风格/评价类型（与网页对齐）
│   └── favorites.js       # 收藏本地存储
└── pages/
    ├── index/             # 首页（核心功能）
    ├── privacy/           # 隐私政策
    ├── disclaimer/        # AI 免责声明
    └── about/             # 工具介绍
```

## 二、本地调试前必做

1. **填写服务器地址**  
   打开 `app.js`，设置：

   ```js
   globalData: {
     apiBase: "https://你的已备案域名",
   }
   ```

   - 勿以 `/` 结尾。  
   - 若 Next 部署在域名根路径，则请求为 `https://域名/api/...`。

2. **HTTPS**  
   必须使用有效 HTTPS 证书（个人测试可用内网穿透+合法证书，正式上架须备案域名）。

3. **微信开发者工具**  
   - 菜单「项目」→「导入项目」，选择本 `miniprogram` 目录。  
   - 将 `project.config.json` 里 `appid` 改为你自己的小程序 AppID（测试可先用测试号）。

4. **开发阶段域名校验**  
   详情 → 本地设置 → 可勾选「不校验合法域名」仅用于本机调试；**真机预览/提审前必须配置合法域名**。

---

## 三、个人主体注册与类目（避坑）

1. **注册**  
   微信公众平台 → 注册「小程序」→ 选择**个人**主体，完成邮箱、微信扫码等流程。

2. **类目**  
   设置 → 基本设置 → 服务类目：建议选择 **工具 → 实用工具**。  
   产品描述可与「通用商家评价回复辅助、文案生成」一致，**避免**在简介/截图/文案中出现「某某平台专属商家」「仅限某某 App」等易被判定为未获授权或混淆官方的字样。

3. **名称与简介**  
   「店评小帮」为通用名称；简介强调**效率工具、文案辅助、本地存储、无账号**，不写诱导分享、付费、外链导流。

4. **用户隐私保护指引**  
   在公众平台「用户隐私保护」中，按实际情况勾选：  
   - 若仅请求你自己的后端域名、使用相册/相机选图、无用户信息接口，按表单逐项声明。  
   - 与小程序内《隐私政策》文案保持一致：**不收集用户个人信息，收藏仅本地**。

---

## 四、服务器域名配置（必过真机）

登录微信公众平台 → 开发 → 开发管理 → 开发设置 → **服务器域名**：

| 类型        | 说明 |
|-------------|------|
| request 合法域名 | 填写你的 API 根域名，如 `https://api.example.com`（无需路径） |
| uploadFile / downloadFile | 本工程未强制要求，按实际补充 |

注意：

- 域名须备案，证书有效。  
- 一个月内修改次数有限，填写前务必确认。  
- 后端需允许小程序请求（无登录场景一般无 Cookie 依赖；注意服务端 CORS 对小程序无意义，但需保证接口可访问）。

---

## 五、与 Next.js 的对接说明

- **识图接口**：`POST /api/extract-review-from-image`  
  Body：`{ "mimeType": "image/jpeg", "imageBase64": "..." }`

- **生成接口**：`POST /api/generate-replies`  
  Body：`{ "review", "category", "style", "ratingType", "customPersona?" }`  
  与网页逻辑一致（品类含自定义、人设可自定义等）。

请将 Next 应用部署到上述 HTTPS 域名；若使用 Vercel/其它平台，把该平台提供的域名加入 request 合法域名。

## 六、上传代码与提审

1. 微信开发者工具中点击「上传」，填写版本号与备注。  
2. 公众平台 → 版本管理 → 提交审核。  
3. 审核信息建议勾选与类目一致的说明，可附：无账号、无支付、无外链、AI 生成仅供参考（与页内免责声明一致）。  
4. 被拒常见原因：类目不符、隐私未配置、简介违规、域名未配置、功能无法使用（接口失败）。按驳回意见逐项修改后再提。

---

## 七、合规与产品红线（摘要）

- 不做登录、注册、手机号、用户画像。  
- 不出现「美团专属/美团商家」等强绑定第三方平台表述（小程序 UI 与说明文档已按通用工具表述）。  
- 无付费、无诱导分享、无 web-view 加载第三方 H5 业务页。  
- 收藏仅 `wx.setStorage`，换机即无数据，需在隐私政策中说明。

---

## 八、后续迭代建议

- 将 `apiBase` 改为「体验版/正式版」分环境（需配合不同小程序或后台配置）。  
- 若审核要求补充「用户协议」，可复制隐私政策结构新增页面并在 `app.json` 注册。  
- 大图识别前可增加压缩（`wx.compressImage`）以降低流量与超时概率。

---

如有接口变更，请同步修改 `utils/api.js` 与网页端 `app/page.tsx` 的请求字段，保持两端一致。
